Dart 언어 기본 문법 및 OOP 개념 요약 정리

1. final과 const의 차이
final은 실행 시점(런타임)에 값이 결정되고, const는 컴파일 시점에 값이 결정된다.
즉, final은 프로그램이 실행될 때 한 번만 값을 정할 수 있고, const는 코드 작성 시점에 미리 확정된 상수여야 한다.

2. 다트의 기본 자료형이 아닌 것
Dart의 기본 자료형은 int, double, String, bool, List, Map, Set 등이 있다.
Character 타입은 존재하지 않으므로 기본 자료형이 아니다.

3. late 키워드의 역할
late는 변수를 나중에 초기화할 때 사용하는 키워드이다.
즉, 변수를 선언만 하고 실제 값은 이후에 할당할 수 있도록 허용한다.
이는 null-safety 기능이 있는 Dart에서 유용하게 사용된다.

4. 다트에서 ? 연산자
?는 변수가 null 값을 가질 수 있음을 나타낸다.
예를 들어 String? name;과 같이 선언하면 null을 허용하는 변수 타입이 된다.

5. 변수를 한 번만 할당하고 변경할 수 없게 만드는 키워드
final 키워드는 한 번 값을 할당하면 이후에 변경할 수 없도록 만든다.
단, 실행 중에 값이 결정될 수 있다는 점에서 const와 다르다.

6. null 값을 허용하는 변수 타입을 나타내는 기호
? 기호를 사용한다.
예: int? number = null; → 이 변수는 null 값을 가질 수 있다.

7. 다트 함수 선언으로 올바른 예시
int add(int a, int b) { return a + b; }
또는 화살표 함수를 사용하여 int add(int a, int b) => a + b; 형태로도 표현할 수 있다.

8. var로 선언한 변수의 특징
var로 선언하면 처음 할당된 값의 타입을 기준으로 타입이 자동 결정된다.
이후에는 다른 타입의 값을 대입할 수 없다.
예: var name = '홍길동'; → name은 String 타입으로 고정된다.

9. 반복문이 아닌 것
if 문은 조건문이지 반복문이 아니다.
Dart의 반복문은 for, while, do-while이다.

10. OOP(객체지향 프로그래밍)의 핵심 개념이 아닌 것
OOP의 핵심 개념은 캡슐화, 상속, 다형성, 추상화이다.
따라서 컴파일은 OOP의 핵심 개념이 아니다.
